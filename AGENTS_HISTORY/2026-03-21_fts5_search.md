# Task: Implement Full-Text Search (FTS5) Support

**Task ID:** `xiiehcor4h4lbdu`

## Objective

Add full-text search support using SQLite's FTS5 module. Create/enhance the list endpoint with a `?search=` parameter. Auto-create FTS indexes for text fields marked as searchable. Rank results by relevance.

### 4. `crates/zerobase-core/src/services/collection_service.rs`
- Added `searchable_fields: Vec<String>` to `CollectionSchemaDto`

### 5. `crates/zerobase-db/src/lib.rs`

### 6. `crates/zerobase-db/src/fts.rs` (new)
- `fts_table_name()` тАФ naming convention: `{collection}_fts`
- `create_fts_index()` тАФ creates FTS5 external content virtual table + triggers + rebuilds
- `drop_fts_index()` тАФ drops FTS table and triggers
- `rebuild_fts_index()` тАФ rebuilds FTS from main table data
- `create_sync_triggers()` тАФ INSERT/UPDATE/DELETE triggers for FTS sync
- `sanitize_search_query()` тАФ escapes FTS5 operators, preserves quoted phrases, strips special chars
- `fts_table_exists()` тАФ checks sqlite_master for FTS table
- **8 unit tests** for sanitization and FTS operations

### 7. `crates/zerobase-db/src/record_repo.rs`
- Rewrote `find_many()` to support FTS5 search:
  - INNER JOINs with FTS table when search is active
  - Combines filter + search WHERE clauses with AND
  - Orders by `rank` (relevance) when no explicit sort provided
  - Falls back gracefully when FTS table doesn't exist
- **8 integration tests** for FTS search: matching, no match, combined filter+search, explicit sort override, pagination, graceful fallback, phrase queries, FTS lifecycle

### 8. `crates/zerobase-db/src/schema_repo.rs`
- `create_collection_impl()`: creates FTS5 index when searchable fields present
- `update_collection_impl()`: drops old FTS index, recreates if searchable fields changed
- `delete_collection_impl()`: drops FTS index before dropping table

### 9. `crates/zerobase-api/src/handlers/records.rs`

  тФФтФА ?search= query param тЖТ RecordQuery.search
