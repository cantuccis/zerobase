# Implement Admin API Endpoint Documentation Page

**Date:** 2026-03-21 15:30
**Task ID:** kdo5bsciybz2j9x

## Summary

Built a full API documentation page for the Zerobase admin dashboard. The page auto-generates endpoint documentation for each collection, showing all available endpoints with methods, URL patterns, request/response examples, filter syntax reference, and copy-to-clipboard functionality.

## What Was Built

### Core Features
- **Collection selector** (sidebar for desktop, dropdown for mobile) showing all collections with type badges
- **Auto-generated endpoint docs** per collection:
  - 5 CRUD endpoints for base/view collections
  - 11 endpoints for auth collections (5 CRUD + 6 auth-specific)
- **Expandable endpoint cards** showing:
  - HTTP method and URL pattern
  - Description and access rule (Public/Superusers only/Custom rule)
  - Query parameters table (for list/view endpoints)
  - Request body examples (for create/update/auth endpoints)
  - Response body examples with realistic field values
  - cURL command examples
- **Fields summary table** showing name, type, required, unique per field
- **Filter syntax reference** with all operators, descriptions, and examples
- **Copy-to-clipboard** on all code blocks
- **Error/loading/empty states** following existing patterns

### Architecture
- Pure function doc generation (`api-docs.ts`) — no side effects, easy to test
- React page component (`ApiDocsPage.tsx`) — follows existing DashboardLayout pattern
- Astro page route (`docs.astro`) at `/_/docs`
- Sidebar navigation entry with document icon
