# URL Field Type Implementation

**Date:** 2026-03-21 15:00
**Task ID:** 8vwp7kw72ecams1

## Summary

Enhanced the existing URL field type with proper validation and domain filtering options (`onlyDomains`, `exceptDomains`), matching the patterns established by the Email field type.

## Changes Made

### Enhanced `UrlOptions` struct
- Added `only_domains: Vec<String>` for domain allow-listing
- Added `except_domains: Vec<String>` for domain block-listing
- Both use case-insensitive comparison

### Improved URL validation (`validate_value`)
- Rejects whitespace anywhere in the URL
- Validates scheme format per RFC 3986 (must start with letter, only alphanumeric/+/-/.)
- Rejects empty content after `://`
- Extracts host from URL authority (handles userinfo, port, path/query/fragment)
- Applies domain allow-list and block-list filtering

### Options validation (`validate`)
- Rejects specifying both `onlyDomains` and `exceptDomains` simultaneously
- Validates that domain entries are non-empty

### Tests (46 URL-specific tests)
- Basic URL format validation (valid schemes, paths, queries, fragments, ports, userinfo)
- Rejection cases (no scheme, empty scheme, invalid scheme chars, whitespace, non-string types)
- Scheme filtering (single, multiple, case-insensitive)
- Domain allow-list (listed/unlisted, case-insensitive, multiple, with port, with userinfo)
- Domain block-list (listed/unlisted, case-insensitive, multiple)
- Options validation (conflicting options, empty entries, defaults)
- Combined scheme + domain filtering
- SQL type and type name assertions
- Serialization round-trip
